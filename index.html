MIL-IMS-LTI

This is the Middlebury Interactive Languages fork of the Instructure IMS-LTI gem. It adds a new outcome data type that allows instructors to have instructor graded assignments within the Middlebury platform on any LMS through LTI.

Problem

Submitting a grade through LTI will have it graded immediately. This prevents an instructor from having a chance to see a list of submissions that need grading and giving each student a grade based on the students work for an essay or audio submission etc.

Solution

Adding a new outcome data extension key/value pair that will allow a submission to be marked as needs grading. This prevents the LMS from grading the assignment and allows the instructor to review it first and give it a grade accordingly.

Usage

First create a tool provider with the OutcomeData extensions:

    tp = IMS::LTI::ToolProvider.new(consumer_key, shared_secret, lti_params)
    tp.extend IMS::LTI::Extensions::OutcomeData::ToolProvider

Next check the lms for support and use post_replace_result_with_data with the new needs_grading param:

    if tp.accepts_outcome_needs_grading? 
        tp.post_replace_result_with_data!(score_lti,'needs_grading' => 'true')
    end

Optionally use url to give the instructor a link to the students work for grading:

    if tp.accepts_outcome_needs_grading? && tp.accepts_outcome_url?
        tp.post_replace_result_with_data!(
            score_lti,
            'needs_grading' => 'true',
            'url' => 'http://www.example.com/assignment/student_id'
        )
    end

Add this gem to your project

$ gem install mil-ims-lti

    # Gemfile
    gem 'mil-ims-lti', '~> 1.1.6'

Requires LMS implementation

Each LMS will need to implement and accept the needs_grading data type then use this value when present to prevent grading from happening on a submission recieved from LTI. This modification has been made to the MIL fork of Canvas as an example.
